New Relic モバイル監視 UI の **Settings** \[設定] の下に、New Relic で監視されているモバイル アプリの変更、アラート条件の編集、およびその他の設定の編集のためのオプションがあります。アプリのデータを認証するには、 [アプリケーション トークンを表示します](/docs/mobile-monitoring/new-relic-mobile/maintenance/configure-settings-for-mobile-monitoring/#application-token)。

<Callout variant="important">
  この機能へのアクセスは、ユーザーの権限によって異なります。
</Callout>

これらの設定を構成するには、 **[one.newrelic.com > All capabilities](https://one.newrelic.com/all-capabilities) > Mobile > (モバイル アプリを選択) > Settings**に移動します。

## アプリケーショントークンを表示する [#application-token]

[データインジェストキー](/docs/apis/intro-apis/new-relic-api-keys) の一つは、当社のモバイル監視アプリケーショントークンです。お客様が監視する各モバイルアプリには、米国とEUのデータセンターの両方で、そのアプリに固有の40文字の16進数文字列が自動的に割り当てられます。このトークンは、お客様のアプリのデータを認証するために必要です。

アプリケーショントークンを表示するには

1. **[one.newrelic.com > All Capabilities](https://one.newrelic.com/all-capabilities)**に移動し、 **Mobile**クリックして、監視対象のアプリを選択します。
2. **Application** または **Installation** を選択します。

<Callout variant="tip">
  モバイルアプリから報告されたデータは、アプリケーショントークンに基づいて、名前の付いたアプリに結び付けられます。単一のアプリ・トークンを持つすべてのアプリは、モバイルデバイスにインストールされたときに、アプリの名前に関係なく、当社のモバイルモニタリングUIの同じ場所にレポートされます。iOS版とAndroid版を分けるために、プラットフォームごとに異なるアプリケーショントークンを使用することをお勧めします。
</Callout>
